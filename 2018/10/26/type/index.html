类型判断到底应该怎么写？

typeof 的问题

typeof 在检查基本类型的时候，typeof null 会返回 object。
typeof 在检查 object 的时候，遇到函数会返回 function，其余的都返回 object。

剩下的我们想要判断是 obj 是哪个类的实例该如何判断呢？

instanceof

obj instanceof Array //返回 true 则是该类的实例。

这样的写法可以判断出实例来。但是有一个弊端，就是在原型链上类，都会返回 true。
[] instanceof Object 也是 true 的。

Object.prototype.toString.call()

调用 Object 上 toString 方法，可以返回他的准确类型。
不过返回值不够优雅。
