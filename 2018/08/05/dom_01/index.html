什么是 DOM？

DOM 是什么

DOM 是 W3C 制定的一套标准，是一组 api 接口。

我们平常使用 js 时，所讲的 DOM 是指的，使用 js 编写的 DOM 接口的具体实现。

因为早先时候标准不完善，所以每个浏览器的具体实现的标准就会有所不同，也就有了 ie，chrome 等浏览器操作 DOM 不兼容的问题。

同时，DOM 接口的具体实现也可以是其他语言来编写的，比如 Dart，Kotlin 等。

DOM 标准分为好几个部分，我们主要关注 DOM CORE，以及 DOM HTML，其他的如 XML，SVG 等暂且不表。

浏览器如何实现 DOM 接口

我们操作 DOM 时，使用最多的对象就是 document 这个对象了，下面我们来看 document 的类型：

document.proto
HTMLDocument {constructor: ƒ, Symbol(Symbol.toStringTag): “HTMLDocument”}

浏览器终端会打印出 document 是 HTMLDocument 的实例。

在 MDN 中：https://developer.mozilla.org/en-US/docs/Web/API/HTMLDocument，可以找到他的继承链。
HTMLDocument->Document->Node->EventTarget

最终我们使用 document 这个对象，来使用这个继承链上的所有方法。

EventTarget 类主要提供了响应事件的能力。

Node 类主要提供节点信息如 NodeType，NodeName 和查找节点 firstChild 的方法。

Document 主要提供修改当前文档模型的方法，以及获取修改文档信息的方法。

HTMLDocument 主要功能是提供 HTML 相关的属性，样式等。

这里可以打印出他们的 prototype 看一下。

浏览器通过以上几个类的继承关系，最终返回给我们一个 HTMLDocument 类型的 document 对象。我们最初通过 document 的来获取，修改我们的 HTML 文档。
