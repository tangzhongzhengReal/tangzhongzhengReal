微信 JSSDK-网页登录，微信浏览器分享

唐忠正
2018-03-28

PC 网页微信登录，微信浏览器分享。

微信登录

首先引入微信登录相关的 js
[http://res.wx.qq.com/connect/zh\_CN/htmledition/js/wxLogin.js]

接着在 html 中创建一个 id 为 aaa 的 div。
在页面加载后调用 wxlogin：

这个东西会创建一个 iframe 放到 aaa 这个 div 上，里面是微信登录的二维码。当扫码成功之后，会执行以下流程：
[] https://images2015.cnblogs.com/blog/731178/201601/731178-20160115132145897-758264873.png 
当用户扫码之后，微信会传递一个 code 到 xxxx.xxx.xx/wxLoginCallback 接口，我们可以使用这个 code，和 appid 以及 AppSecret 来跟微信换取 accesstoken 和 openid
access_token*➕ openid 就可以换取用户的基本信息了。
当获取成功之后，xxxx.xxx.xx/wxLoginCallback 可以直接做一个重定向，如果重定向的 url 是同域的，这样网页就可以重定向跳转了。
同时这里也可以用 websocket，接口成功之后通过 websocket 来通讯，由前端来控制跳转等操作，更多样化。

微信内网页分享，vue 单页应用

  1. 判断是否是微信浏览器，如果是再引入 jssdk
    开始判断有没有 weixinjsbridge 这个对象，如果有，那么肯定就是在微信中，如果没有就去监听这个对象 ready 的事件
  1. 引入 js
  1. 最重要的一点就是签名。
    这里要分享的链接地址，一定要与签名相匹配，才会成功。而这个签名对于 ios 和 android 来讲是不同。
    android 中，所用的签名 url 是当前分享网页的 url。而 ios 中，所用签名的 url 是这个网站第一次所请求签名的那个 url。
  1. 二次分享的问题：从转发的链接进入网页后，再次分享会不成功，原因在于转发出来的链接会包含一些微信的参数，请求签名会失败。
    可以在路由之前做一遍判断，如果有那么就去掉，用新的 url 强制刷新。
