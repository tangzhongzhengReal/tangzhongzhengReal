唐忠正

FLUTTER, WEB

关于 flutter 的组件的一些初步理解和总结。

Weight 的分类

stateless 和 stateful 两种组件的不同点在于，在组件内部需不需要一个维持组件逻辑的状态，必须是内部需要这个变量，组件件传递进来的变量不算。

组件中需要接受的参数都在 weight 类中用 final 标记的变量来接受，因为组件都是不可变的。所以他的类属性只能赋值一次。

Statefulweight 组件需要一个 createdState()函数来返回 build 的对象。

proxyweight，代理组件传递数据的（没用过，以后细说）

renderobjectweight：直接关于渲染的（没用过

state 的生命周期

initState 初始的时候会调用一次，开着调试模式修改的话可能不执行

一个神奇的地方：
当传递参数变化时，构造函数和 didUpdateWidget 都会执行，但是 initState 不会执行。 （下一篇解释）

移除时会调用 deactivate，dispose 方法。

还有一个钩子叫 didChangeDependencies ，这个东西我不懂。
