响应式原理

众所周知，Vue 的响应式原理是 Object.defineProperty 实现的。Object.defineProperty 实现响应式，其实没啥，就是每个数据对应着，一系列使用他的 node。当数据更新，重新设定这个 node 信息。这样就实现了当数据改变，dom 也跟着改变了。

所以，就有一下几步要做：

  1. 给 data 的每个值去设置 Object.defineProperty
  2. 每个值都有一个储存对应节点的一个数组，
  3. 在合适的时间，将对应节点放入到这个数组中。
  4. 当 set 的时候去 循环这个数组，更新信息。

不足之处

这样写，每个 data 对应的 节点依赖，都会放到 vm 的 dep 中，每一个 data 中值改的时候，都会执行所有的 watcher，这样既占了内存又多了计算。

改进的话有亮种方式：

  1. 采用低粒度的更新，就是把每个 data 的值对应的 watcher 放到各自对应的数组中，当 data 的一个值修改时，只执行其对应的 watcher 更新。（这样就是 vue1.0 的方法）
  2. 采用中等粒度的更新，就是向现在这样将所有的值都放到 vm 的 dep 中，但是 watcher 里也只放 vm 的更新函数，有值更新，vm 对应的 dom 就去更新就完了。如果其中再加入虚拟 dom 的 diff 算法，拿这就是 vue2.0 了！
